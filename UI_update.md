# ALLMusic UI 现代化与功能对齐计划

本计划旨在通过“先修剪、再接线、后重构”的策略，解决项目中四个 UI 版本（V4.2 Glam, V8 Clay, V8 Fallout, Y2K AF）存在的冗余展示与功能断层问题，提升应用专业感。

---

## 1. 核心调整方针
1.  **功能零容忍**：所有可见按钮必须绑定有效 API。无法实现的按钮必须隐藏或转为禁用状态。
2.  **单一事实来源**：同一状态（如播放进度、连接状态）全屏仅保留一处展示，避免视觉干扰。
3.  **响应式优先**：移除手动预览模型（如手机壳），利用 Flex/Grid 实现真正的自适应布局。

---

## 2. 各版本具体调整方案

### V8 Clay (目标：打造标准桌面端 UI)
*   **清理动作**：
    *   移除右侧边栏的 `now-card`（Now Playing）和手机预览模型。
    *   移除顶部独立的“清空”按钮。
*   **修复对齐**：
    *   将右下角 `btn-quality` (HQ) 接入 `bridge.setQuality`。
    *   为左下角 `user-chip` 增加登出/切换账号菜单。
*   **空间再分配**：
    *   **宽度重塑**：中间歌曲列表区域从原来的固定比例扩展至 `flex-grow`，占据侧边栏移除后的空间。
    *   **沉浸歌词**：将歌词从右侧小方块改为点击底部播放栏封面后弹出的“全屏毛玻璃歌词层”。

### V8 Fallout (目标：打造主题化极客 UI)
*   **清理动作**：
    *   移除伪 CSS 频谱分析仪 (`.viz`)。
    *   合并搜索栏右侧重复的 `Reload` 按钮。
*   **修复对齐**：
    *   修复歌词提示语，直接渲染 `bridge` 返回的实时歌词。
    *   将 `tactic-btn.primary` 样式统一应用到所有交互确认动作上。
*   **空间再分配**：
    *   **列表加长**：利用原本频谱分析仪占据的垂直空间，延长歌曲列表的显示行数。

### V4.2 Glam (目标：怀旧经典模式)
*   **清理动作**：
    *   移除登录页 Stepper 引导。
    *   移除控制栏的 “✨” 与 “DISCO” 装饰按钮。
*   **修复对齐**：
    *   将 `♥ 收藏` 按钮接入 `bridge.toggleLike` 并实现状态回显。
    *   移除“批量收藏”等未实现功能，避免用户误操作。

- [x] **V8 Fallout**: 删除 `.viz` 能量条。

### 第二阶段：逻辑“接线” (P1)

- [x] **Like**: 为 V4.2 接入收藏状态同步逻辑。
- [x] **Auth**: 为各版本账号区域增加“点击重新登录”的跳转逻辑。

- [x] **V8 Clay**: 重新计算中间容器宽度，确保歌曲列表在 1080p 下有更好的横向伸展。
- [x] **Overlay**: 实现一个跨版本共用的“全屏沉浸歌词层”，作为 V8 版本移除侧边栏后的功能补偿。

## 4. 验证清单 (UAT)
1.  **无死角点击**：屏幕上所有看起来像按钮的元素点击后均有反馈。
2.  **状态唯一性**：在 V8 Clay 下，不存在两个显示相同歌名的区域。
3.  **响应式验证**：拉伸窗口时，布局平滑过渡，不再出现遮挡歌曲列表的冗余侧栏。
